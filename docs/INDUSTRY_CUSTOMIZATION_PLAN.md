### قاعدة التصنيف

```
هل عندك منتج مادي تبيعه؟
├── نعم → هل تصنعه؟
│   ├── نعم → MF (تصنيع)
│   └── لا → هل تبيعه بالقطعة للأفراد؟
│       ├── نعم → هل هو طعام؟
│       │   ├── نعم → FB (مطاعم)
│       │   └── لا → هل هو دواء/مستلزمات طبية؟
│       │       ├── نعم → PH (صيدليات)
│       │       └── لا → هل تبيعه أونلاين فقط؟
│       │           ├── نعم → EC (تجارة إلكترونية)
│       │           └── لا → RT (تجزئة)
│       └── لا → WS (جملة وتوزيع)
├── خدمة فقط → هل هي صيانة + قطع غيار؟
│   ├── نعم → WK (ورش)
│   └── لا → هل هي مشروع بمدة وتكاليف؟
│       ├── نعم → CN (مقاولات)
│       └── لا → هل هي نقل/شحن؟
│           ├── نعم → LG (لوجستيات)
│           └── لا → SV (خدمات مهنية)
└── زراعة/مخزون حي → AG (زراعة)
```

### منطق كل نشاط جديد

#### 💊 PH — صيدليات
- **POS:** ✅ بيع مباشر في الصيدلية (كاشير)
- **Stock:** ✅ أدوية + تواريخ صلاحية + دفعات (batch tracking)
- **Manufacturing:** ❌
- **Projects:** ❌
- **Services:** ❌
- **خصوصية:** تتبع الصلاحية (batch/expiry) — ميزة موجودة في نظام المخزون

#### 🔧 WK — ورش
- **POS:** ✅ فاتورة فورية للعميل (خدمة + قطع غيار)
- **Stock:** ✅ قطع غيار ومستلزمات
- **Manufacturing:** ❌ (لا خطوط إنتاج)
- **Projects:** ❌
- **Services:** ✅ طلبات صيانة + تتبع التكاليف
- **خصوصية:** الفاتورة = خدمة + قطع مستخدمة

#### 🛒 EC — تجارة إلكترونية
- **POS:** ❌ (لا محل فعلي — البيع عبر فاتورة عادية أو ربط خارجي)
- **Stock:** ✅ مستودع شحن
- **Manufacturing:** ❌
- **Projects:** ❌
- **Services:** ❌
- **خصوصية:** بوليصة الشحن + تتبع الطلبات + طرق دفع إلكتروني

#### 🚛 LG — لوجستيات
- **POS:** ❌
- **Stock:** ✅ بضائع العملاء في المستودعات
- **Manufacturing:** ❌
- **Projects:** ❌
- **Services:** ✅ طلبات الشحن كطلبات خدمة
- **خصوصية:** إيراد النقل ≠ بيع بضاعة — فاتورة خدمة

#### 🌾 AG — زراعة
- **POS:** ❌ (البيع بالجملة غالباً)
- **Stock:** ✅ محاصيل + أعلاف + مخزون حي (مواشي/دواجن)
- **Manufacturing:** ❌ (الإنتاج الزراعي ≠ تصنيع صناعي — لا BOM)
- **Projects:** ❌
- **Services:** ❌
- **خصوصية:** مخزون بيولوجي (IAS 41) + تكاليف موسمية + إهلاك أشجار

## 3. شجرة الحسابات لكل نشاط

### الفلسفة المحاسبية

**القاعدة:** كل نشاط يحصل على شجرة حسابات أساسية مشتركة + حسابات متخصصة حسب طبيعة عمله.

**الترقيم المتبع (SOCPA / IFRS):**

```
1xxxx = الأصول (Assets)
  11xxx = أصول متداولة (Current Assets)
  12xxx = ذمم مدينة (Receivables)
  13xxx = مخزون (Inventory)
  15xxx = مصاريف مدفوعة مقدماً (Prepaid)
  16xxx = أصول ثابتة (Fixed Assets)
  17xxx = أصول غير ملموسة (Intangible)
  18xxx = إهلاك متراكم (Accumulated Depreciation)

2xxxx = الالتزامات (Liabilities)
  21xxx = التزامات متداولة (Current Liabilities)
  22xxx = التزامات طويلة الأجل (Long-term)

3xxxx = حقوق الملكية (Equity)

4xxxx = الإيرادات (Revenue)
  41xxx = إيرادات النشاط الرئيسي
  42xxx = إيرادات أخرى

5xxxx = تكلفة المبيعات / الإنتاج (COGS / Cost of Sales)

6xxxx = المصاريف التشغيلية (Operating Expenses)
  61xxx = رواتب وأجور
  62xxx = إيجارات
  63xxx = مرافق
  64xxx = تسويق
  65xxx = مصاريف إدارية

7xxxx = مصاريف أخرى / مالية

8xxxx = حسابات ضريبية
```

async def seed_industry_coa(db, industry_key: str, replace_existing: bool = False):
    """
    يزرع الحسابات الأساسية + المتخصصة لنوع النشاط.
    
    Args:
        db: database session (company-specific)
        industry_key: مفتاح النشاط (retail, restaurant, etc.)
