## 8. التوصيات

1. **تفعيل pg_trgm (أولوية قصوى)**: `CREATE EXTENSION IF NOT EXISTS pg_trgm;` وإنشاء GIN indexes على الأعمدة النصية الرئيسية:
   ```sql
   CREATE INDEX idx_products_name_trgm ON products USING GIN (product_name gin_trgm_ops);
   CREATE INDEX idx_parties_name_trgm ON parties USING GIN (name gin_trgm_ops);
   ```
   هذا يحوّل `ILIKE '%x%'` من Full Table Scan إلى Index Scan. تنفيذ 5 دقائق — تأثير هائل.

2. **تفعيل PostgreSQL Full-Text Search**: إضافة عمود `search_vector tsvector` لكل كيان رئيسي، مع GIN index و trigger للتحديث التلقائي:
   ```sql
   ALTER TABLE products ADD COLUMN search_vector tsvector;
   CREATE INDEX idx_products_fts ON products USING GIN (search_vector);
   CREATE TRIGGER trg_products_fts BEFORE INSERT OR UPDATE ON products
   FOR EACH ROW EXECUTE FUNCTION tsvector_update_trigger(search_vector, 'pg_catalog.arabic', product_name, product_code, description);
   ```

3. **بناء Unified Search API**: `GET /search?q=...` نقطة واحدة تبحث في العملاء + المنتجات + الفواتير + الموظفين دفعة واحدة، مع ranking و pagination و branch scoping.

4. **إصلاح `duplicate_detection.py`**: تنظيف أرقام الهاتف قبل التخزين (عمود `phone_clean`) بدلاً من REGEXP في WHERE.

5. **وضع `LIMIT` صارم على نتائج البحث**: بحد أقصى 50 نتيجة لكل كيان.

6. **تسجيل استعلامات البحث في سجل التدقيق**: لأغراض أمنية وتحليلية (ماذا يبحث المستخدمون؟).

7. **إضافة Autocomplete**: `GET /search/suggest?q=...` باستخدام `similarity()` من pg_trgm.

8. **فهرسة محتوى المستندات**: استخدام `pg_textparser` أو خدمة خارجية (Tika) لاستخراج نص PDF/Word وجعله قابلاً للبحث.

### **الدرجة الإجمالية: 37/100 (غير صالح للاستخدام مع بيانات حقيقية)**

> **الخلاصة**: "محرك البحث" في AMAN ERP ليس محرك بحث. هو `ILIKE '%query%'` على جداول PostgreSQL بدون فهارس نصوص. هذا يعمل بقبول مع 1000 سجل لكنه **سينهار مع 50,000 منتج أو 10,000 عميل**. البحث الشامل (Ctrl+K) ليس بحثاً في البيانات بل قائمة صفحات ثابتة.
>
> الخبر السعيد: الإصلاح **بسيط وسريع ومجاني**. PostgreSQL لديه `pg_trgm` مدمج (لا حاجة لخادم Elasticsearch منفصل). خمس دقائق من DDL تحوّل أداء البحث من "دقائق" إلى "ميلي ثانية":
>
> ```sql
> CREATE EXTENSION pg_trgm;
> CREATE INDEX ON products USING GIN (product_name gin_trgm_ops);
> -- هذا كل شيء. ILIKE '%x%' سيصبح Index Scan بدلاً من Full Table Scan
> ```
>
> تكلفة الإصلاح: صفر. تأثيره: تحويلي.
