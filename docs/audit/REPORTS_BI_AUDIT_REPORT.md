1. **إصلاح التحليل الأفقي**: تحويله من `n × m` استعلامات إلى استعلام واحد مجمع: `SELECT account_id, period, SUM(...) GROUP BY account_id, period WITH period defined by CASE WHEN`.

2. **إضافة عمود `cash_flow_classification` لجدول الحسابات**: قيم `operating` / `investing` / `financing`. يُملأ عند إنشاء الحساب ويُستخدم في تقرير IAS 7 بدلاً من الاستدلال النصي.

3. **إضافة عمود `classification` لجدول الحسابات**: قيم `current_asset` / `fixed_asset` / `current_liability` / `long_term_liability` لدعم النسب المالية بدقة دون الاعتماد على ترقيم الحسابات.

4. **تحسين أداء الميزانية وقائمة الدخل**: استبدال `LEFT JOIN ... (je.id IS NULL OR ...)` بـ `LEFT JOIN LATERAL (SELECT ... WHERE ...)` لتمكين استخدام الفهارس.

5. **إضافة فهارس أداء**: `CREATE INDEX ON journal_lines(account_id, journal_entry_id)` و `CREATE INDEX ON journal_entries(entry_date, branch_id, status)`.

6. **توسيع مقارنة الفترات**: دعم `periods` لأكثر من فترتين مع حساب التغير لكل فترة مقابل سابقتها (سلسلة تطور)، وليس فقط بين الأولى والثانية.

7. **ترقية الكاش إلى Redis مشترك**: لتجنب تكرار الحساب عبر العمال المتعددين.
