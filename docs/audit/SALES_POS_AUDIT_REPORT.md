## ملخص الخلل حسب الخطورة

### 🔴 P0 — حرج (خطر مالي أو قانوني)
1. **ZATCA: لا يوجد تدفق clearance إلزامي** — الفواتير تُنشأ دون موافقة ZATCA Phase 2
2. **ZATCA: لا توجد إدارة CSID** — الشهادات لا تُجدد تلقائيًا

### 🔴 P1 — عالي (ثغرة أمنية أو فقدان بيانات)
3. **صلاحية إلغاء الفاتورة `sales.create` فقط** — أي كاشير يمكنه إلغاء أي فاتورة مؤكدة (`invoices.py:756`)
4. **لا يوجد API لتعديل الفواتير** — أي خطأ يتطلب إلغاء وإعادة إنشاء
5. **POS: المبيعات غير المتصلة لا تُعالج أبدًا** — `pos_offline_inbox` بدون worker (`governance.py:1210`)
6. **ZATCA: فشل التجهيز لا يمنع إنشاء الفاتورة** — `try/except` صامت (`invoices.py:667`)
7. **POS: لا يوجد كشف تعارض للمزامنة** — المخزون قد يتغير أثناء عدم الاتصال
8. **صلاحية المرتجعات `sales.create`** — لا تمييز بين المسودة والاعتماد (`returns.py:92,224`)
9. **صلاحية الإشعارات الدائنة `sales.create`** — يجب أن تتطلب صلاحية محاسبية (`credit_notes.py:150`)

### 🟠 P2 — متوسط
10. POS: الخصم العام بعد الضريبة (`pos.py:431`) — مخالف لـ ZATCA
11. POS: العروض/الكوبونات لا تُطبق تلقائيًا على الإجمالي (`pos.py:397-487`)
12. عكس المخزون عند الإلغاء: لا يتحقق من وجود سجل المخزون (`invoices.py:807`)
13. البحث عن القيد المحاسبي عند الإلغاء: يستخدم `reference` بدل `source+source_id` (`invoices.py:820`)
14. فشل تحديث رصيد العميل في POS لا يُبلغ المستخدم (`pos.py:756`)
15. مفاتيح ZATCA الخاصة مخزنة نصًا واضحًا (`utils/zatca.py:299`)
16. POS: مفاتيح ربط حسابات مختلفة عن فواتير المبيعات (`pos.py:626`)
