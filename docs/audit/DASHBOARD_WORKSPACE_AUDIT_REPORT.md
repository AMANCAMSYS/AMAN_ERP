## 8. التوصيات

1. **توحيد مصدر بيانات المبيعات**: جعل `get_dashboard_stats` تجمع المبيعات من `invoices + pos_orders` بنفس منطق تقرير المبيعات، أو استخدام نفس الدالة الداخلية (`_get_profit_loss_data`). لا يجوز أن يرى المدير رقمين مختلفين.

2. **بناء نظام Smart Alerts**: إنشاء جداول `alert_rules` (عتبة + شرط + إجراء) و `alert_history`. إضافة مجدول يفحص القواعد كل 5 دقائق ويُرسل إشعارات (منخفض المخزون، متأخرات السداد، تجاوز حدود، إلخ).

3. **تفعيل KPI Alerts**: ربط دوال `_build_executive_alerts` و `_build_financial_alerts` بمجدول الإشعارات.

4. **إصلاح تضارب COGS**: إعادة تسمية `profit` إلى `gross_profit` أو احتساب COGS من GL (حسابات 5xxxx) وطرحها.

5. **إصلاح pending_tasks**: إضافة فلتر الفرع (`branch_id`) إلى جميع استعلامات المهام المعلقة.

6. **تحسين cash_status**: استخدام عتبات حقيقية مثل: إذا `cash < avg_monthly_expenses` → "خطر"، إذا `cash < 3 * avg_monthly_expenses` → "منخفض".

7. **إضافة مؤشر حداثة البيانات**: عرض `_cache_meta` في استجابة الـ Dashboard، مع طابع زمني لآخر تحديث.

8. **إصلاح الأخطاء الصغيرة**: `invoice_type = 'sale'` → `'sales'`، فلتر `overdue` في صفحة الفواتير، Lazy Loading للـ widgets.
